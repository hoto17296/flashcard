# フラッシュカードアプリ 仕様書

## 概要

記憶学習を助けるためのフラッシュカードアプリ。  
複数の「カード集」を用意し、1枚ずつカードをめくりながら問題と答えを確認できる。

## 対象ユーザー・動作環境

- **ユーザー:** 本人または小学生の子供
- **端末:** スマートフォン（縦向き）またはタブレット
- **アーキテクチャ:** フロントエンドのみの SPA（バックエンドなし）

---

## データ構造

### カード集 (Card Set)

1カード集 = 1つの JSON ファイルとして管理する。  
JSON ファイルはリポジトリ内の所定のディレクトリ（例: `app/src/data/`）に配置する。

```json
{
  "id": "alphabet",
  "title": "アルファベットの読み",
  "order": 1,
  "tags": ["アルファベット"],
  "description": "A〜Zのアルファベットの読み方を覚えよう",
  "cards": [
    { "question": "A", "answer": "エー" },
    { "question": "B", "answer": "ビー" }
  ]
}
```

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | ○ | カード集の一意な識別子 |
| `title` | string | ○ | カード集の名前（画面に表示） |
| `order` | number | ○ | カード集一覧画面での表示順（昇順） |
| `tags` | string[] | − | カード集のタグ（日本語可。一覧画面のフィルタリングに使用。省略可） |
| `description` | string | − | カード集の説明文（カード集トップ画面に表示。省略可） |
| `cards` | array | ○ | カードの配列 |
| `cards[].question` | string | ○ | 問題のテキスト |
| `cards[].answer` | string | ○ | 答えのテキスト |

### カード集の登録

JSON ファイルを `app/src/data/` に配置するだけで自動的に一覧に追加される（`import.meta.glob` による自動検出）。ソースコードの編集は不要。

## 画面構成・フロー

```
[カード集一覧画面]
      ↓ カード集を選択
[カード集トップ画面]
      ↓ 「スタート」ボタンを押す
[カード表示画面]
      ↓ 最後のカードを見終わる
[カード集トップ画面] ← 戻る
```

---

### 画面 1: カード集一覧画面

- アプリ起動時に最初に表示される画面
- 利用可能なカード集をリスト形式で表示する（`order` 昇順）
- タグが存在する場合、タグフィルターボタンを表示する
  - タグをタップすると該当カード集だけに絞り込む
  - 同じタグを再度タップすると絞り込みを解除する
- カード集をタップすると「カード集トップ画面」へ遷移する

---

### 画面 2: カード集トップ画面

- 選択されたカード集の名前・説明文・カード枚数を表示する
- カードの表示順を選択できる
  - **「順番通り」:** 登録された順番で表示
  - **「シャッフル」:** ランダムな順番で表示
- 「スタート」ボタンを押すと「カード表示画面」へ遷移する
- カード集一覧に戻るボタンを表示する

---

### 画面 3: カード表示画面

- 現在のカード番号と総枚数を表示する（例: `3 / 10`）
- **問題表示フェーズ**
  - カードの「問題」のみを表示する
  - 「答えを見る」ボタンを表示する
- **答え表示フェーズ**（「答えを見る」ボタンを押した後）
  - カードの「問題」と「答え」を両方表示する
  - 「次のカードへ」ボタンを表示する
- **最後のカード**
  - 「次のカードへ」ボタンを押すと「カード集トップ画面」へ戻る
- 「← 戻る」ボタンを表示し、途中でも「カード集トップ画面」へ戻れる

---

## 機能スコープ

### 含まれるもの

- カード集の一覧表示・タグによるフィルタリング
- カードを1枚ずつめくる操作
- 順番通り / シャッフルの切り替え
- スマートフォン・タブレット向けのレスポンシブレイアウト

### 含まれないもの（将来対応）

- カード・カード集の作成・編集・削除機能（アプリ内）
- 学習ログ・正解率の記録
- ユーザー認証
- データの永続化・クラウド同期
